# Plan Round 2 — Pages publiques (post-review)

## Approche

5 pages frontend independantes, toutes dans app/(frontend)/. Toutes sont des Server Components
qui chargent les donnees via getPayload(). Pas de DB requise au build (ISR + fallback statique).
Les pages utilisent des donnees statiques/hardcodees pour le moment (les collections sont vides,
le contenu sera ajoute par l'admin via /admin une fois PostgreSQL configure).

## Conventions pour TOUS les agents

- Server Components par defaut, "use client" SEULEMENT si interactivite (formulaire, filtre, menu)
- Metadata SEO (title, description) dans chaque page.tsx
- shadcn/ui pour tous les composants UI
- Tailwind CSS, JAMAIS de CSS modules
- Mobile-first responsive
- Fichiers < 150 lignes, un composant par fichier
- Donnees hardcodees pour le moment (les collections sont vides)
- Images : utiliser des divs placeholder avec bg-zinc-800 (pas d'images reelles)
- Icones : lucide-react
