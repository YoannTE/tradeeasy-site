## Phase 1 — Setup (sequentiel, bloquant)

### Agent "setup" (general-purpose)

**Tache** : Creer le projet Next.js 15.4 + Payload CMS 3.x + PostgreSQL + shadcn/ui

1. `npx create-next-app@15.4 . --typescript --tailwind --eslint --app --src-dir=false --import-alias="@/*"`
2. Installer Payload CMS 3.x manuellement (payload, @payloadcms/next, @payloadcms/db-postgres, @payloadcms/richtext-lexical)
3. Creer `.env.local` avec :
   - `DATABASE_URL=postgresql://...` (PostgreSQL locale ou Neon)
   - `PAYLOAD_SECRET=simplifypro-dev-secret-change-in-prod`
4. Creer `.env.example` (meme structure, valeurs vides)
5. Configurer `payload.config.ts` avec PostgreSQL adapter
6. Configurer `next.config.ts` avec `withPayload`
7. Configurer `tsconfig.json` avec alias `@payload-config`
8. Creer les fichiers admin dans `app/(payload)/` :
   - layout.tsx (handleServerFunctions avec config + importMap)
   - admin/[[...segments]]/page.tsx
   - admin/importMap.js
   - api/[...slug]/route.ts
   - custom.scss
9. Installer shadcn/ui : `npx shadcn@latest init`
10. Configurer theme sombre dans globals.css :
    - Background: zinc-950 (#09090B), Primary: blue-500 (#3B82F6)
    - Surface: zinc-900 (#18181B), Border: zinc-800 (#27272A)
11. Installer Inter font via `next/font/google`
12. Creer `app/layout.tsx` MINIMAL : html lang="en", body avec font Inter, class="dark"
13. Creer dossier `media/` a la racine
14. Verifier que `npm run dev` fonctionne

**Fichiers a lire** : `.project/design.md`, `CLAUDE.md`, `BRIEF.md`

## Phase 2 — Collections + Layout + Seed (parallele, 2 agents)

### Agent "collections" (general-purpose)

**Tache** : Creer 7 collections Payload. PAS de localized: true (anglais uniquement au MVP).

1. **Users** (etendre collection native) :
   - Champs : firstName, lastName, tradingviewUsername (text, unique, required),
     tradingviewAccessStatus (select: pending/granted/revoked, default pending),
     tradingviewAccessGrantedAt (date), stripeCustomerId (text),
     referralCode (text, unique), referredBy (relationship Users),
     lastLoginAt (date)
   - Access : admin read/update all, user read/update own only
   - Hook beforeChange : auto-generer referralCode via crypto.randomUUID().slice(0,8) si vide
   - Hook afterLogin : update lastLoginAt

2. **Subscription** (collection separee) :
   - Champs : user (relationship Users, required), type (select: monthly/annual),
     status (select: trial/active/payment_failed/cancelled/expired),
     startDate (date), trialActivatedAt (date),
     trialEndDate (date), nextRenewalDate (date),
     stripeSubscriptionId (text), stripeEventId (text),
     promoCodeUsed (relationship PromoCode), lastSyncedAt (date)
   - Champs Stripe-synced en admin readOnly : status, type, startDate, nextRenewalDate, trialEndDate
   - Access : admin read all, user read own only
   - Hook beforeChange : empecher creation si un abonnement actif/essai/echec_paiement existe deja pour le user

3. **Media** : upload standard, imageSizes : thumbnail (400x300), card (800x600), avatar (200x200 fit cover)

4. **Video** : title (text), description (textarea), youtubeUrl (text, required),
   category (select: installation/usage/strategy/trading_live), displayOrder (number), publishDate (date)
   Access : read public, write admin only

5. **Testimonial** : clientName (text), role (text), content (textarea), rating (number 1-5),
   avatar (upload Media), date (date)
   Access : read public, write admin only

6. **TradeScreenshot** : image (upload Media, required), title (text), description (textarea),
   asset (text ex: "EUR/USD"), result (text ex: "+120 pips"), tradeDate (date)
   Access : read public, write admin only

7. **PromoCode** : code (text, unique, required), discountType (select: percentage/fixed_amount),
   discountValue (number, required), appliesTo (select: monthly/annual/both),
   duration (select: once/repeating/forever), validFrom (date), validUntil (date),
   active (checkbox, default true), maxUses (number), currentUses (number, default 0, admin readOnly)
   Access : read admin only (PAS public), write admin only
   Note : currentUses sera incremente atomiquement au Round 3 (raw SQL)

**Fichiers a lire** : `.project/app.md` (section Entites + Regles metier), `CLAUDE.md`
