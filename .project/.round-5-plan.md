# Plan Round 5 — Espace abonne & Emails

## Approche

Round 5 est un mix frontend (pages abonne) + backend (emails, cron, Stripe Portal).
2 agents en parallele :

- Agent "dashboard" : pages frontend abonne (attente + espace perso + Stripe Portal)
- Agent "emails" : templates email + envoi + cron job

L'email "Acces accorde" est DEJA fait au Round 4 (hook + template). On ne le refait pas.

## Agent "dashboard" — Pages abonne

### Tache 1 : Page d'attente post-inscription — app/(frontend)/onboarding/page.tsx

Page affichee apres inscription, en attendant l'acces TradingView.
Protegee par requireAuth().

Composants dans `components/onboarding/` :

**onboarding-page.tsx** (Server Component) :

- Recupere l'utilisateur via getUser()
- Recupere son statut TradingView (tradingviewAccessStatus)
- Si status === "granted" → redirect vers /dashboard
- Sinon affiche le contenu d'attente

Structure de la page :

- Titre "Welcome to SimplifyPro!" text-3xl font-bold
- Sous-titre "Your TradingView access is being set up. Here's how to prepare." text-zinc-400
- Badge statut : "Access Pending" (bg-amber-500/10 text-amber-400) ou "Access Granted" (bg-green-500/10 text-green-400)

**onboarding-checklist.tsx** (Server Component) :

- Checklist de preparation :
  1. "Create a TradingView account" (lien externe tradingview.com)
  2. "Set up your chart workspace"
  3. "Watch the installation tutorial" (lien vers /videos)
  4. "Join our Discord community" (lien Discord)
- Chaque item : icone check ou cercle vide, titre, description courte

**onboarding-video.tsx** (Server Component) :

- Embed video YouTube placeholder : div bg-zinc-900 rounded-xl h-[300px] avec icone Play
- Titre "How to Install SimplifyPro V6.0"

Layout : max-w-3xl mx-auto, py-20, espacement genereux

### Tache 2 : Espace personnel — app/(frontend)/dashboard/page.tsx

Page protegee par requireAuth(). C'est le hub de l'abonne.

Composants dans `components/dashboard/` :

**dashboard-page.tsx** (Server Component) :

- Recupere user via getUser()
- Recupere subscription via Payload (find where user=userId, sort by -createdAt, limit 1)
- Si pas de subscription → redirect vers /pricing

**subscription-card.tsx** (Server Component) :

- Carte principale bg-zinc-900 border border-zinc-800 rounded-xl p-6
- Titre "Your Subscription"
- Statut avec badge colore : Trial (amber), Active (green), Payment Failed (red), Cancelled (zinc)
- Type : Monthly $49/mo ou Annual $349/year
- Date renouvellement : "Next renewal: March 30, 2026"
- Si trial : "Trial ends: March 30, 2026" + jours restants

**tradingview-status.tsx** (Server Component) :

- Carte bg-zinc-900
- "TradingView Access" titre
- Badge statut : Pending (amber), Granted (green), Revoked (red)
- Pseudo TradingView affiche
- Si Pending : "Your access will be granted within 12 hours"
- Si Granted : "Your indicator is active on TradingView"

**dashboard-links.tsx** (Server Component) :

- Grille de liens rapides (2x2 ou 3 colonnes) :
  - "Videos & Tutorials" → /videos (icone PlayCircle)
  - "Join Discord" → lien Discord externe (icone MessageCircle)
  - "Manage Billing" → /dashboard/billing (icone CreditCard)
  - "Contact Support" → /contact (icone Mail)

**dashboard-actions.tsx** (Client Component "use client") :

- Bouton "Manage Billing" → POST /api/stripe/create-portal-session → redirect vers Stripe Portal
- Bouton "Cancel Subscription" avec confirmation dialog (AlertDialog shadcn)
- Installer : `npx shadcn@latest add alert-dialog`

### Tache 3 : Stripe Customer Portal — app/api/stripe/create-portal-session/route.ts

API route POST :

- Verifie que l'utilisateur est authentifie
- Recupere le stripeCustomerId du user
- Si !isStripeConfigured() : retourner { url: '/dashboard', mock: true }
- Sinon : appeler stripe.billingPortal.sessions.create({ customer, return_url })
- Retourner { url: session.url }

Le frontend redirige vers cette URL.

Fichiers a lire : lib/stripe/stripe.ts, lib/auth/get-user.ts, collections/Users.ts

### Tache 7 : Cron job rappel essai — app/api/cron/trial-reminder/route.ts

API route GET (appelee par Vercel Cron ou manuellement) :

- Verifier un secret token (CRON_SECRET env var) pour securiser l'endpoint
- Chercher toutes les Subscriptions avec :
  - status = "trial"
  - trialActivatedAt defini
  - trialEndDate dans 2 jours (entre J+5 et J+6 apres activation)
- Pour chaque : envoyer l'email trial-reminder au user
- Retourner { sent: N, errors: N }

Ajouter CRON_SECRET dans .env.local et .env.example.

Fichiers a lire : lib/email/send-email.ts, lib/email/templates/ (existants), lib/stripe/webhook-handlers.ts

## Conventions

- requireAuth() pour proteger les pages abonne
- getUser() pour recuperer l'utilisateur dans les Server Components
- overrideAccess: true pour les queries Payload dans les API routes
- isStripeConfigured() pour mock en dev
- Fichiers < 150 lignes, un composant par fichier
- shadcn/ui (AlertDialog, Badge)
- Templates email : inline CSS, fond sombre, pas de dependances externes
