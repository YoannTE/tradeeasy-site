# Plan Round 7 — Systeme multilingue EN/FR

## Approche

Utiliser next-intl pour la gestion des traductions cote frontend.
Pas de routing base sur la locale (/en/, /fr/) — on utilise un cookie pour stocker
la preference de langue. Ca evite de changer toutes les URLs existantes.

3 agents en parallele :

- Agent "i18n-setup" : installation next-intl, config, middleware, provider, fichiers JSON, selecteur de langue
- Agent "translate-public" : traduire toutes les pages publiques + navbar + footer + SEO
- Agent "translate-auth-email" : traduire les pages auth + protegees + templates email

IMPORTANT : Les agents "translate" dependent de l'agent "setup" (les fichiers JSON doivent exister).
Donc : Phase 1 = setup, Phase 2 = traductions en parallele.

Pattern pour les Server Components :

```tsx
import { getTranslations } from "next-intl/server";
export default async function MyComponent() {
  const t = await getTranslations("sectionName");
  return <h1>{t("title")}</h1>;
}
```

Pattern pour les Client Components :

```tsx
"use client";
import { useTranslations } from "next-intl";
export function MyComponent() {
  const t = useTranslations("sectionName");
  return <h1>{t("title")}</h1>;
}
```
