## Agent "infra" — Rate limiting, Cron, Deploy config, Responsive

### Tache 4 : Rate limiting

Creer `lib/rate-limit.ts` :

- Simple in-memory rate limiter (Map avec timestamps)
- Fonction rateLimit(ip: string, limit: number, windowMs: number): boolean
- Retourne true si OK, false si limite atteinte
- NOTE : en serverless (Vercel), le Map est reset a chaque cold start. C'est un rate limiting "best effort" acceptable au MVP.

Appliquer dans `app/api/auth/signup/route.ts` :

- Au debut du handler : extraire IP (x-forwarded-for ou request headers)
- Si rateLimit(ip, 5, 3600000) === false → retourner 429 "Too many requests"
